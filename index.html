🔄 Quick Fixes — Try These First

This is the #1 fix for most display problems and blank pages.

Windows Press Ctrl + Shift + R
Mac Press Cmd + Shift + R
Mobile Close the browser tab completely, then re-open the link.

This forces your browser to download a fresh copy instead of using a stored (cached) version.
Browser extensions (especially ad blockers and privacy tools) can interfere with apps.

Chrome Press Ctrl + Shift + N (Windows) or Cmd + Shift + N (Mac)
Firefox Press Ctrl + Shift + P
Safari File → New Private Window

If the app works fine in Private mode, an extension is the cause. Try disabling them one at a time.
If a force-reload didn't help, a full cache clear usually does.

Chrome / Edge Press Ctrl + Shift + Delete → select Cached images & Cookies → Clear data
Firefox Press Ctrl + Shift + Delete → same steps
Safari (Mac) Safari menu → Settings → Privacy → Manage Website Data → Remove All

Note: Clearing browser data will also clear your Reading Journal entries if you have any saved locally. Make sure to use the Share / Export feature first to back up your list.
If the problem persists, try opening the app in a different browser:

  • Google Chrome (recommended — best compatibility)
  • Microsoft Edge
  • Firefox
  • Safari (Mac / iPhone)
If it works in one browser but not another, that tells us exactly where to look.

🐛 Common App Problems

This is almost always caused by one of two things:

1. A JavaScript error crashed the app. Press F12 → click the Console tab → look for red error text. If you see one, copy it and paste it into the App Issue tab above.

2. A browser extension is blocking scripts. Try Incognito mode (see above). Ad blockers and script blockers are the most common culprits.

Quick fixes to try first: force-reload (Ctrl + Shift + R), then try incognito mode, then try a different browser.
Step 1: Check your internet connection — try loading another website.
Step 2: Force-reload: Ctrl + Shift + R (Windows) / Cmd + Shift + R (Mac)
Step 3: Try the direct GitHub link instead of the custom domain (check the app's documentation page for the backup URL).
Step 4: Try on a different device or network (e.g., switch from Wi-Fi to mobile data).

If none of these work, the app may be temporarily down. Submit an App Issue report so Rachel can investigate.
RLB apps store your data in your browser's local storage — it stays on your device and is never sent to a server. This means:

  • Data is device-specific — it won't appear on a different device
  • Clearing browser data or cookies deletes it
  • Private/Incognito mode does not save data between sessions
  • Some browsers auto-clear storage after a period of inactivity
Prevention: Use the Share / Export feature in the Reading Journal regularly to create a backup URL you can bookmark.
